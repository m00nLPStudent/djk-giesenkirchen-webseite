# Component Library

## Ziel

Wiederverwendbare Komponenten sollen zentral dokumentiert werden, damit keine doppelten UI-Bausteine entstehen.

## Admin UI

- `EntityCard` für Karten in Übersichten
- `EntityBadge` für Status und Kategorien
- `StatisticGrid` für Kennzahlen
- `TabNavigation` für Formular-Reiter
- `AdminRemoveButton` für sichere Löschdialoge

## Admin Forms

- `InputField`
- `TextareaField`
- `SelectField`
- `EmailField`
- `PhoneField`
- `SortOrderField`
- `ActiveStatusField`
- `FormSection`
- `FormGrid`
- `FormActions`

## Website UI

- Navigation-Komponenten
- News-Karten
- Mannschaftsbereiche
- Profil-Komponenten für Spieler und Trainer

## Regel

Vor jeder neuen Komponente prüfen, ob ein vorhandener Baustein erweitert werden kann.
