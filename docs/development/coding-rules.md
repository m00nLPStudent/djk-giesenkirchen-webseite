# Coding Rules

Zentrale Entwicklungsregeln.

## Kurzregeln

- Erst analysieren, dann gezielt ändern.
- Feature-Verhalten bei Refactoring nicht verändern.
- Gemeinsame Logik in src/lib auslagern, wenn Wiederverwendung klar ist.
- Änderungen mit Diff nachvollziehbar halten.
- Dokumentation bei strukturellen Änderungen im selben Arbeitsschritt aktualisieren.
